# Contributing to fsquared-ai projects

Thanks for contributing. We value small, high-quality PRs and clear issues.

## Where to start
- Browse open issues in the target repo
- Use issue templates for bug reports, feature requests, or story pitches

## PR guidelines
- Keep PRs focused (one intent)
- Update docs when behavior changes
- Prefer deterministic changes and reproducible outputs
- Add sample inputs/outputs when relevant

## Naming conventions (Everduck)
- Episode IDs: `ED-YYYYMMDD-###-slug`
- Release manifests should reference the episode ID and include output links

## Review process
- Maintainers will request changes via PR comments
- We may ask you to split large PRs into smaller ones
